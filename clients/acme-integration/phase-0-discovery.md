## Problem Statement

Acme Logistics operates a trade compliance and customs brokerage platform built on Node-RED. Their engineering team manually builds integration flows in the Node-RED editor — each flow takes 3+ hours to wire, test, and deploy. With 200+ flows in production across 6 tenants and a backlog of 40+ new flows requested by operations, the team cannot keep pace.

Manual flow building introduces inconsistency: different engineers wire error handling differently, credential management varies by flow, and there is no automated testing — flows are validated by visual inspection in the editor and manual trigger testing.

## Proposed Solution

AI-assisted flow generation with programmatic deployment via the Node-RED Admin API.

1. **Generate** — AI produces flow JSON from a structured specification (natural language description of trigger, logic, and output)
2. **Validate** — automated checks against the Node-RED runtime (node types exist, wires are valid, credentials present)
3. **Deploy** — push agent sends flow JSON to staging via Admin API
4. **Test** — Playwright runs integration tests against the deployed flow
5. **Promote** — on test pass, same JSON is deployed to production tenant

Target: reduce flow creation from 3+ hours to under 10 minutes (including review and testing).

```
└────────┬────────┘
         │
         ▼
┌──────────────────┐
│ Compliance Check  │  Denied party screening, HTS code validation,
│ (Node-RED flow)   │  restricted items check
└────────┬─────────┘
         │ (CLEARED only)
         ▼
┌─────────────────┐
│ Customs Filing    │  Assemble documents, calculate duties, submit
│ (Node-RED flow)   │  to customs broker API
└────────┬─────────┘
         │
         ▼
┌─────────────────┐
│ Status Tracking   │  Aggregate events from all flows into unified
│ (Node-RED flow)   │  shipment timeline, push to ERP
└─────────────────┘
```

Each box is a separate Node-RED flow. Flows communicate via MQTT messages and HTTP callbacks. The status tracking flow subscribes to events from all other flows.

## Scope Boundaries

### In Scope

- AI-assisted flow generation from specifications
- Programmatic flow deployment via Node-RED Admin API
- Playwright integration test suite for all four core flows
- Push agent for staging → production promotion
- Documentation of Node-RED API gotchas and patterns

### Out of Scope

- Migration off Node-RED (not requested, existing investment too large)
- Changes to custom node implementations (maintained by client's core team)
- Multi-tenant management UI (client has existing admin panel)
- EDI format changes (using client's existing EDI parser nodes)
- Customs regulation updates (compliance rules managed by operations team)

## Risks

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| Node-RED Admin API changes between versions | Low | High | Pin Node-RED version. Test API behavior on upgrade in staging first. |
| AI-generated flows with subtle logic errors | Medium | High | Playwright integration tests catch execution-order bugs that visual inspection misses. |
| Editor rendering differences for AI-generated flows | High | Low | Functional, not visual. Document that AI flows look different in the editor. Train team on this distinction. |
| Credential management via API is fragile | Medium | Medium | Validate credential nodes exist before deploy. Never store credentials in flow JSON — use Node-RED credential store. |
| MQTT message loss during redeploy | Medium | Medium | Status tracking flow hydrates from PostgreSQL on startup. Acceptable data delay of 15-30s during deploy window. |

## Success Criteria

1. New flow creation time reduced from 3+ hours to under 10 minutes
2. All four core flows pass Playwright integration tests on every deploy
3. Push agent successfully deploys to staging and production with rollback capability
4. Zero compliance violations caused by flow logic errors (denied party screening must always run before downstream processing)
5. Engineering team can review AI-generated flow JSON and approve deploys without needing to understand the AI generation process
